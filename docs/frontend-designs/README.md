# Front-end Designs

The designs have been generated for your GitHub Campaigns for Bulk PRs desktop web application:

- Campaigns Dashboard: The main dashboard view displays active and historical campaigns, their status, and key metrics, with options for filtering, sorting, and creating new campaigns.

- New Campaign/Spec Editor: This screen features a large, syntax-highlighting text editor for defining campaign specifications in YAML, offering validation, preview, and save functionalities.

- Campaign Details View: A detailed view for a selected campaign, showing its specification, run history, overall status, and controls to manage the campaign and its drift policies.

- Repository Targets View: This screen lists all repositories targeted by a campaign, showing their selection status, reasons, and providing options to retry failed selections.

- Run Details View: Provides a deep dive into a specific campaign run, displaying its status, logs, and access to generated artifacts like diff patches or summary files.

- Changesets/Pull Request View: A consolidated view of all pull requests generated by a campaign, showing key information and enabling bulk actions across various repositories.

- Audit Log View: A comprehensive screen displaying the audit trail of all user and system actions with timestamps, actors, action types, and correlation IDs for full traceability.
